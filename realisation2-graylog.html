Réalisation professionnelle n°2

Mise en place d'une solution centralisée de gestion des logs Graylog

Objectif : assurer le maintien en conditions opérationnelles et détecter les incidents de sécurité grâce à la centralisation des journaux OPNsense.

Retour à la page E6 Télécharger la fiche

Présentation de la réalisation

Objectif

Déployer Graylog afin de collecter, centraliser, rechercher et analyser les logs de sécurité générés par le pare-feu OPNsense.

Périmètre technique

  • VM Debian 12 hébergée sur Proxmox
  • Graylog Server 5.x
  • MongoDB pour les métadonnées
  • OpenSearch pour l'indexation
  • Syslog UDP sur le port 514

Étapes de réalisation

1 Déploiement du serveur Graylog

Installation d'une VM Debian 12 dans le VLAN Admin avec une adresse IP fixe : 10.20.210.10.

2 Installation de la pile logicielle

Installation de Graylog Server, MongoDB et OpenSearch puis vérification du démarrage des services.

  • Interface web Graylog : http://10.20.210.10:9000
  • Services contrôlés avec systemctl

3 Création de l'input Syslog

Création d'un input Syslog UDP sur le port 514 afin de recevoir les logs envoyés par OPNsense.

4 Configuration OPNsense vers Graylog

Configuration de l'envoi des journaux OPNsense vers Graylog pour récupérer les événements de filtrage, les connexions bloquées et les activités réseau.

5 Dashboards et alertes

Création de tableaux de bord pour visualiser les messages reçus, les sources principales, les connexions bloquées et les événements par VLAN.

  • Dashboard Vue générale
  • Dashboard Sécurité
  • Dashboard VLANs
  • Alerte e-mail sur volume de connexions bloquées

6 Tests de validation

Génération de trafic bloqué depuis le VLAN invité puis vérification de la réception des logs et du déclenchement des alertes.

Résultats obtenus

Logs centralisés

Les journaux OPNsense sont regroupés dans Graylog pour faciliter les recherches et l'analyse.

Dashboards sécurité

Les tableaux de bord permettent de visualiser les connexions bloquées, les top IPs et l'activité par VLAN.

Alertes

Une notification peut être déclenchée en cas d'événements suspects ou de volume anormal de blocages.

Analyse d'incidents

Les logs facilitent le diagnostic après un incident réseau ou une tentative d'accès non autorisée.
