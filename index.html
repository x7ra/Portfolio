Veille informatique

Sujet

Vulnérabilités critiques et sécurisation des infrastructures réseau. Ma veille porte sur les failles de sécurité qui peuvent toucher des serveurs, pare-feu, VPN, équipements réseau, services web ou partages de fichiers.

Problématique

Comment suivre les failles importantes pour protéger une infrastructure réseau et réagir rapidement ?

Méthode

1. Flux RSS

Je consulte les flux RSS du CERT-FR.

2. Vulnérabilités

Je repère les vulnérabilités critiques.

3. Vérification

Je vérifie si mon infrastructure est concernée.

4. Recommandations

Je consulte les recommandations officielles.

5. Action

Je mets en place une mise à jour, une règle firewall, une supervision ou une alerte.

Exemple concret

Exemple : faille sur un serveur de fichiers

Si une faille touche un serveur de fichiers, les utilisateurs peuvent perdre l’accès à leurs documents ou un attaquant peut tenter d’exploiter le service. Grâce à ma veille, je peux recevoir l’alerte, vérifier si mon infrastructure est concernée, appliquer les correctifs, limiter les accès avec OPNsense, surveiller les logs avec Graylog et vérifier la disponibilité avec Zabbix.
